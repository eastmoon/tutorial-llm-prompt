# 註解

大語言模型沒有像程式語言那樣的原生註解語法，但有幾種經過驗證的技巧可以讓模型忽略或不處理特定內容。

## XML 註解標籤

模型在訓練資料中大量接觸 HTML/XML，對此標籤有較強的「忽略」傾向：

```xml
<!-- 這段是給提示詞設計者看的備註，模型不需處理 -->
請分析以下程式碼的效能問題。

<!-- TODO: 未來加入安全性檢查的指令 -->
<!-- 測試版本 v0.3，尚未正式啟用多語言支援 -->
```

優點：最接近原生語義，效果最穩定。

## 明確指令框架

直接告訴模型哪些區塊要忽略，語義最清晰：

```
<instructions_for_llm>
  請回答使用者的問題，並忽略所有 <dev_note> 標籤內的內容。
</instructions_for_llm>

<dev_note>
  這個提示詞用於客服場景，A/B 測試組 B，
  上線日期：2026-04，負責人：Jacky
  尚未驗證繁體中文邊界案例
</dev_note>

請用友善的語氣回答使用者的退款問題。
```

## 反模式

以下寫法應該避免使用

```
// 這是備註，請忽略（❌ 程式語言風格，模型不一定忽略）

# 內部備註：這個提示詞還在測試中（❌ Markdown 標題，會被當成指令讀取）

（備註：以下為實驗性功能）（❌ 括號語義模糊，模型可能納入回應）
```
