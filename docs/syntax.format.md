# 格式

格式在程式語言中用於輸出排版 ( 例如 printf )，對用於產生統一輸出訊息至關重要，在用於生成內容的大語言模型更為重要

要讓大語言模型穩定輸出特定排版格式，核心思路是「明確宣告 + 結構示範 + 負面約束」三位一體。

## 明確宣告輸出格式

在提示詞開頭直接聲明格式規則，不要讓模型自行判斷。

```
請以下列格式回應，不得偏離：
- 標題使用 ## 層級
- 每個段落不超過 3 句
- 重點以條列呈現，不使用段落敘述
```

## 提供具體範例（Few-shot）

這是最有效的手段，直接給「樣板」讓模型學習邊界：

```
請參考以下格式回答：

【範例輸入】：什麼是 Docker？
【範例輸出】：
**定義**：Docker 是容器化平台。
**用途**：
- 環境隔離
- 快速部署
**一句總結**：Docker 讓應用跑在一致的容器中。

---
現在請回答：什麼是 Kubernetes？
```

## 負面約束（禁止清單）

光說「要什麼」不夠，要明確說「不要什麼」：

```
格式規則：
- 禁止使用 Markdown 標題（#）
- 禁止出現「當然！」「好的！」等開場白
- 禁止在答案前後加入解釋或補充說明
- 只輸出格式內容，不加任何前言
```

## 強化語（Anchoring Words）

在格式指令前加上強化語，提升遵守率：

+ 普通：```請使用以下格式```
+ 強調：```務必嚴格遵循以下格式```
+ 最高；```無論如何，只能輸出以下格式，不得增減任何內容```
