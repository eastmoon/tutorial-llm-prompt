# 迴圈

大語言模型沒有原生的迴圈執行引擎，迴圈的本質是：明確告知模型「重複的對象」、「每次迭代的動作」、「終止條件」三個要素；若缺少任何一個要素，模型可能：

+ 只處理第一個項目就停止
+ 無限展開直到 Context 耗盡
+ 跳過某些項目或合併處理

## 自然語言句型

以下為自然語言的迴圈句型，這樣的寫法簡單，但條件邊界模糊，並未明確說明迴圈三個要素。

```
有一個矩陣 NUMBERS，其中包括 10 個隨機整數，在 NUMBERS 中的數字為 X，則依據規則 ```Y = X * X``` 計算 Y，並輸出至 RESULT。
```

## 結構化迴圈

結構化是利用定義符號，來確保每個句子執行的意義，從而符合前述的迴圈要素。

```
<!-- 定義迴圈符號 -->
## 定義

**[ITERATE_OVER]**：要迭代的集合或範圍
**[FOR_EACH]**：每次迭代對單一項目執行的動作
**[UNTIL]**：終止條件（處理完畢 / 達到數量 / 條件成立）
```

### For-Each

ForEach 是最常見的迴圈結構，執行次數依據矩陣數量。

```
## 變數

ITEM = [markdown | json | plain]

## 任務

**[ITERATE_OVER]** `$ITEMS` 清單中的每一個項目

**[FOR_EACH]** 單一項目執行以下步驟：
1. 標示項目編號（第 N 項，共 M 項）
2. 解釋內容
3. 提供使用時機

**[UNTIL]** `$ITEMS` 清單中所有項目皆處理完畢

**[FORMAT]** 每個項目之間以分隔線 `---` 區隔
```

### For Range

For Range 依據指定數量執行。

```
## 變數

START_INDEX = 1
FOR_RANGE = 5

## 任務

**[ITERATE_OVER]** 從 `$START_INDEX` 到 `$START_INDEX + $FOR_RANGE - 1`

**[FOR_EACH]** 當前計數值 `$I`：
- 在每個項目開頭標示 `[$I/$FOR_RANGE]`
- 說明一件 `1900 + $I` 的歷史事件

**[UNTIL]** `$I` 達到 `$START_INDEX + $FOR_RANGE - 1`
```

### While

While 是常見的條件終止形式迴圈。

```
## 變數

CONTENT = 內容描述
CRITERIA = [檢查條件1, 檢查條件2]
MAX_ROUNDS = 3

## 任務

**[ITERATE_OVER]** `$CONTENT` 的精煉過程

**[FOR_EACH]** 每輪迭代 `$ROUND`（從第 1 輪開始）：

1. 標示當前輪次：`[第 $ROUND 輪 / 最多 $MAX_ROUNDS 輪]`
2. 對照 `$CRITERIA` 逐條檢查 `$CONTENT`
3. 列出未通過的條件
4. 輸出修正後的 `$CONTENT`
5. 重新檢查所有 `$CRITERIA`

**[UNTIL]** 滿足以下任一條件即停止：
- 所有 `$CRITERIA` 全部通過 → 輸出 `✅ 精煉完成`
- `$ROUND` 達到 `$MAX_ROUNDS` → 輸出 `⚠️ 已達最大輪次，列出仍未通過的條件`
```

以 While 迴圈來說，最重要是定義終止條件，若條件撰寫錯誤，會導致無窮迴圈的現象。

也由於終止條件的描述精確度直接影響模型是否會提前停止或過度執行，對於迴圈終止條件的寫法強度可參考如下：

```
<!-- ❌ 弱：模型可能自行判斷何時停止 -->
**[UNTIL]** 處理完所有項目

<!-- ✅ 中：明確指定集合耗盡 -->
**[UNTIL]** `$ITEMS` 清單中無剩餘項目

<!-- ✅ 強：數量 + 狀態雙重確認 -->
**[UNTIL]** 滿足以下「全部」條件才停止：
- `$ITEMS` 清單已無剩餘項目
- 每個項目皆已輸出對應的處理結果
- 已輸出完整的總結區塊

<!-- ✅ 強：有安全上限防止無限迴圈 -->
**[UNTIL]** 以下任一條件成立即停止：
- 所有項目處理完畢
- 迭代次數達到 `$MAX_ROUNDS`（安全上限）
- 偵測到連續兩輪輸出完全相同（收斂條件）
```
