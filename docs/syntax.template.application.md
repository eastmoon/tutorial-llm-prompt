# 樣板運用

關鍵字驅動樣板擴展內容應用 ( Keyword-Driven Template Expansion Content Application )，其核心概念如下：

```
[原始需求段落]
    ↓ 提煉
[關鍵名詞清單]
    ↓ 映射
[標題骨架（Template Slots）]
    ↓ 展開
[完整文件細節]
```

這套設計的核心優勢是：關鍵字是需求的語義錨點，標題繼承了這個錨點，細節只能在錨點語義範圍內展開，從而大幅減少文件的幻覺偏移問題。

## 單一提示詞

在單一文件中構成內容。

```markdown
你是一位規格文件設計師。

## 輸入
以下是使用者的需求段落：
<requirement>
{{USER_INPUT}}
</requirement>

## 執行步驟
**Step 1 — 提煉關鍵名詞**
掃描需求，識別：名詞性實體、動詞性操作、修飾性屬性。

**Step 2 — 構成標題骨架**
將關鍵名詞組合為文件章節標題，遵循此命名模式：
- 功能類：`{實體} {動作}機制`
- 規則類：`{屬性} 規範定義`
- 關係類：`{實體A} 與 {實體B} 的互動邊界`

**Step 3 — 基於標題展開細節**
對每個標題，輸出：
- **概述**：本章節解決的問題
- **規格條目**：條列式規範（以「應」、「不得」、「當...時」為開頭）
- **範例**：至少一個具體情境說明

## 輸出格式
以 Markdown 輸出完整規格文件，包含目錄。
```

## 兩段式提示詞

在這文件結構中，依據步驟先提煉、後展開。

### Phase 1 - 關鍵字提煉指令

```markdown
## 任務：關鍵字提煉

從以下需求描述中，識別並分類關鍵名詞：

<requirement>
{{USER_INPUT}}
</requirement>

請依下列類別輸出關鍵字：
- **實體（Entity）**：系統、模組、角色名稱
- **動作（Action）**：核心操作與流程
- **屬性（Attribute）**：限制條件、規格、規則
- **關係（Relation）**：實體間的依賴與互動

輸出格式：
```json
{
  "entities": [],
  "actions": [],
  "attributes": [],
  "relations": []
}
```
```

### Phase 2 - 標題映射與細節生成

```markdown
## 任務：基於關鍵字生成規格文件

給定以下關鍵字分類：
<keywords>
{{KEYWORDS_JSON}}
</keywords>

依照下列模板結構，將關鍵字填入標題，並基於標題語義展開細節描述：

### 模板規則
- 每個 Entity 生成一個「## {Entity} 規格」章節
- 每個 Action 在對應 Entity 章節下生成「### {Action} 行為定義」
- Attribute 作為各章節的「限制條件」子項
- Relation 生成「## 依賴關係圖」章節

### 細節展開標準
對每個標題，展開：
1. **定義**：一句話說明此標題涵蓋的範圍
2. **前置條件**：觸發或存在的條件
3. **行為描述**：具體規範（3–5 條）
4. **邊界與例外**：不處理的情況
```

## Constitution 驅動

若基於 Spec-Driven Development 概念，則可參考 GitHub Spec Kit 類的框架，將規則抽離為 `constitution.md` 文件。

```markdown
# constitution.md — 標題生成規範

## 關鍵字提煉規則
1. 只提取名詞與動名詞，排除連接詞與形容詞
2. 同義詞合併（例：「使用者」「用戶」→ `User`）
3. 專有名詞保持原文大小寫

## 標題構成模式
| 關鍵字類型 | 標題模板 | 範例 |
|-----------|---------|------|
| 單一實體 | `{Entity} 定義與邊界` | `訂單 定義與邊界` |
| 實體+動作 | `{Entity} 的 {Action} 流程` | `用戶 的 登入 流程` |
| 屬性規則 | `{Attribute} 約束規範` | `權限等級 約束規範` |

## 細節展開規範
- 每節必含：定義、前提、規則條目、例外處理
- 規則條目使用 RFC 2119 語氣詞（MUST / SHOULD / MAY）
- 細節不得超出關鍵字語義範圍（防止幻覺擴散）
```

主要提示詞則會調用樣板並規範其他必要資訊。

```markdown
依照 constitution.md 的規則，從以下需求提煉關鍵字並生成規格文件：

<requirement>{{USER_INPUT}}</requirement>
```
